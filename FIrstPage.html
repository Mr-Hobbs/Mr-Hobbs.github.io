POSITION

The Position property specifies where in the document an element will go. It can be fine-tuned from there with the top, right, bottm, left, and z-index properties.

  1. position: static
    • The default value for position!
    • Element fits in the normal flow of the document
    • top, right, bottom, left, and z-index have no-effect
  2. position: relative
    • Elements fit in the normal flow of the document, BUT...
    • top, right, bottom, left, and z-index can be used to scoot the element from its default spot
    • Even if the element is offset, its original space in the document is maintained
  3. position: absolute
    • Element is removed from normal document flow, and is now relative to nearest parent element with an explicit position property
    • If no parent has a position property other than default (static), it's relative to the root container
    • No space is left where it used to be!
    • top, right, bottom, left, and z-index can be used to scoot the element further
  4. position: fixed
    • Element is removed from normal document flow, and is positioned relative to root container element
    • NO space is left where it used to be
    • top, right, bottom, left, and z-index can be used to scoot the element relative to the root element
    • When you scroll, it will stay in place!
  5. position: sticky

z-index

Specifies the elevation of an element relative to other elements. THe lower the z-index the farther back/deeper in the stack it is.

  • z-index: 0; is under...
  • z-index: 1; is under...
  • z-index: 2; is under...
  • z-index: 3;

Flexbox!

  • Flexbox is a layout method for laying out items in rows or columns
  • It is a relatively recent addition to CSS, created to address the difficulty of creating layouts
  • Flexbox allows us to distribute space dynamically across elements of an unknown size, hence the term "flex"
  • Main axis, cross axis
  • Flex Direction
  • Flex Wrap
  • justify-content: flex-start; --- this is the default option. Your content is at the LEFT of the container
  • justify-content: flex-end; --- Your content is at the RIGHT of the container
  • justify-content: center; --- distributes the empty space on the left and right oof the content
  • justify-content: space-between; --- The content touches the edge of the container and space is ditributed evenly between the elements. GOOD FOR NAVBARS
  • justify-content: space-around; --- evenly adds space even on the left and right of the left and right-most elements.
    • All of the above also works for columns as well
  • Order of items to do Flex stuff. All done in the outer container:
    1. display: flex;
    2. flex-direction: row; (or column, or row-reverse or column-reverse)
    3. justify-content: start; (or flex-start or flex-end or center or space-between or space-around or space-evenly)
  • align-items
    1. align-items is used to align elements along the cross access (USUALLY VERTICALLY; perpendicular to the main access).
    2. align-items: flex-start
    3. align-items: center
    4. align-items: flex-end
    5. align-items: stretch (THIS IS THE DEFAULT)
    6. align-items: space-around
    7. align-items: center
  • align-content
    1. align-content requires "flex-wrap: wrap;" be enabled in your larger container in order to take effect.
    2. align-content: space-between; -- controls the space between columns or rows
  • align-self
    1. align-self is the same context as align items but we assign align-self to an individual flex item. Target one box or element in a container.
    2. align-self: flex-end;
    3. align-self: flex-start;
    4. etc.

When you want to center something in the middle of a container do: justify-content: center; and align-items: center;.

Additional Flex Properties

These apply to elements INSIDE the Flex Container

  • Flex
    • flex: is shorthand for flex-grow, flex-shrink, & flex-basis
    • It sets how a flex item will grow or shrink to fit the space available in its flex container
  • flex-grow
    • Dictates how much of the available space inside the flex container the item should take up
    • flex-grow: 1
    • flex-grow: 2
      • This element would take up twice as much empty space as the other element
  • flex-shrink
    • Sets the flex shrink factor of a flex item. If the size of all flex items is larger than the flex container, items shrink to fit according to flex-shrink.
      • flex-shrink: 1
      • flex-shrink: 3
        • This element shrinks 3 times faster than the other
  • flex-basis
    • flex-basis sets the initial main size of a flex item. It sets the size of the content box unless otherwise set with box-sizing (width/height)
    • If flex-direction: row then flex-basis sets the width. If flex-direction: column then flex-basis sets the height
  • Review flex on Udemy
  • Order Flex Item
    • It is a property to apply to a flex item to change the order that the elements are sorted by in that container.
  • Flex box is good for navbars. For full pages, using b CSS Grid is better (we learn this later)

Media Queries

  • Beginner's guide to media queries
  • Media queries allow us to specify styles that are only applied at certain device screen widths. They are the lifeblood of developing finely-tuned responsive websites!
  • @media initiates a media query, and allows us to follow up by specifying other important properties
  • only is a keyword called a logical operator that prevents older browsers that don't support media queries from applying these styles. Otherwise, an older browser would apply these styles in general (and not just for the specified screen width)
  • screen is the media type in this case. THere are other media types like print or all, but screen is what helps us write styles for different device screen sizes.
  • and is another logical operator that helps us combine media features and/or media types together. In this case, we're combining a media type (screen) and a media feature (the max-widthportion).
  • if you have conflicting media query styles the one that is lowest in order in the css file takes precidence
  • max-width is actually just the width media feature with max added on. There are many other media features, which can also be modified by min and max
  • THE SETUP FOR A MEDIA QUERY LOOKS SOMETHING LIKE THIS
    • @media only screen and (max-width: '768px') {
      put normal css code goes here
      }
  • in the section above we see AND used. We can use other logical operators (or, not)

  • Common Breakpoints in screen widths

    • Mobile Breakpoint: 0-480px
    • Tablet Breakpoint: 481-768px
    • Desktop (Small) Breakpoint: 769-1024px
    • Desktop (Large) Breakpoint: 1025px - up
    • NOTE: THese breakpoints are a general rule of thumb, but in reality every unique layout and design requires attention to its own specific breakpoints. Test layouts thoroughly!

  • Layout Design Types

  • Desktop-first is a design/development approach in which you build a website to work on desktop, and then you'd make it responsive for smaller devices and screen sizes after.
    In the earlier days of the web before mobile web browsing became widely accessible thanks to smary phones, most websites were not responsive.
    As mobile usage grew, sites that were already optimized for desktop began to add mobile responsiveness.

  • Mobile-first is a design philosophy that emphasizes designing/building the mobile experience for a website first, and then adding in layout customizations for larger viewport sizes after.
    As the internet matured and web traffic shifted to be more mobile-centric with the adoption of smart-phones, this philosophy grew in popularity.
    Web designers and developers needed to ensure that the good experience for their websites carried over to smaller screens and devices other than desktop.

CSS Grid

  • CSS Grid Layout
  • Grid is a layout system built into CSS!
    • A collection of CSS properties used to manage layouts far easier than before
  • A CSS Grid is primarily made up of a Grid Container with Grid Items inside of it.
    • Grid Containers are parent elements while Grid Items are children of the parent
  • HOW IS IT DIFFERENT FROM FLEXBOX?
    • Flexbox = row OR column
    • Grid = row AND column
  • Grid Containers are just parent elements set to display either grid or inline-grid
  • Containers can define columns and rows (or Grid Tracks) using grid-template-columns + grid-template-rows
  • fr stands for "fraction" - it represents an even proportion of the grid, which is useful
    when we don't want to calculate % widths
  • Containers can use shorthand grid-template, which accepts row values / column values all in one!

  • minmax() lets you define a minimum and a maximum value for the width a column should span or the height a row should span
    • grid-template-columns: minmax(100px, 1fr);
  • .container {
    display: grid;
    grid-template-columns: 1fr 1fr;
    grid-template-rows: 1fr 2fr 1fr 2fr;
    }
  • repeat() lets you define many columns or rows with certain values with ease, instead of typing the same thing over and over
    • grid-template-columns: repeat(4, 1fr);
    • COMBINATION
      • grid-template-columns: repeat(5, minmax(100px, 250px));
  • fit-content() tells a GRID TRACK to be as wide/tall as its content, but with a max value that it shouldn't exceed.
    It takes a bunch of different values, like px, % etc. as a max.
    • grid-template-rows: fit-content(50px) 1fr 1fr
  • Grid Containers come with gap styles to easily control the spacing between Grid Items, such as column-gap and row-gap
    • display: grid;
      grid-template: 1fr 1fr / repeat(4, 1ft);
      column-gap: 20px;
      row-gap: 30px;
      OR USE grid-gap: 20px;
  • Review module 227 for notes here. WORKING WITH GRIDLINES

  • Positioning Grid Items

  • Using number values, you can specify which grid lines the Grid Item should span. It will start at the beginning of the column/row
    you specify and end right before the column/row you specify
    • .container .item {
      grid-column-start: 1;
      grid-column-end: 3;
      grid-row-start: 1
      grid-row-end: 2;
      }
  • Using span, you can specify how many columns/rows a Grid Item should span. Note: by default it starts in the first column/row, and the number refers to grid tracks, not grid lines.
    • .container .item {
      grid-column-start: 1;
      grid-column-end: span 2;
      grid-row-start: 1
      grid-row-end: span 2;
      }
  • You can use grid-column & grid-row as a shorthand. It takes a start value / end value.
    • .container .item {
      grid-column: 1 / 3;
      grid-row: 1 / 3
      }
  • You can also name your lines in a Grid Container! This can be helpful later when we position Grid Items, since we
    can specify which line they start at and which they end at (more on that soon).
    • .container {
      display: grid;
      grid-template-columns: [col1-start] 1fr [col1-end col2-start] 1fr [col2-end col3-start] 1fr [col3-end];
      grid-template-rows: [row1-start] 1fr [row1-end row2-start] 1fr [row2-end];
      }
  • Using name values, you can specify what grid lines the Grid Item should span.
    • .container .item {
      grid-column-start: col1-start;
      grid-column-end: col2-end;
      grid-row-start: row1-start;
      grid-row-end: row2-end; }

  • grid-template-areas:

  • You can use grid-template-areas as a shorthand way to setup a grid in a really visually-friendly way. Visualize the items in the grid in code an voila!
    • .container {
      grid-template-areas:
      "a a c"
      "b b c";
      }
    • Next, simply name your grid items by matching their grid-area name, and watch as they get positioned exactly where you specified!
      • .container .item1 {
        grid-area: a;
        }

        .container .item2 {
        grid-area: b;
        }

        .container .item3 {
        grid-area: c;
        }

  • Using Grid-Template-Areas with Rows and Columns

    • .container {
      height: 700px;
      width: 700px;
      margin: 200px auto;
      background-color: black;
      box-shadow: 0 6px 20px rgba(227, 183, 183, 0.2);
      display: grid;
      grid-template-columns: 1fr 1fr 150px;
      grid-template-rows: 100px 1fr 100px;
      grid-template-areas:
      "navbar navbar navbar"
      "main main sidebar"
      "footer footer footer";
      }

      .box {
      color: white;
      font-size: 1.5rem;
      text-align: center;
      }
      .first {
      grid-area: navbar;
      }

      .second {
      grid-area: main;
      }

      .third {
      grid-area: sidebar;
      }

      .fourth {
      grid-area: footer;
      }

  • You can use grid-auto-flow to tell the grid how/where to add new items in the grid. It defaults to row.
    • .container {
      display: grid;
      grid-auto-flow: row;
      }

  • You can use grid-auto-flow to tell the grid to add new items to fill up each column first instead
    • .container {
      display: grid;
      grid-auto-flow: column;
      }

  • Adding the dense keyword after row or column tells the grid to pack in the space with the next available item that would fit, in order.
    Yes column dense is also a thing
    • .container {
      display: grid;
      grid-auto-flow: row dense;
      }

  • When grid-auto-flow: is set to rows or columns and you have more .items than you've alotted space for in the grid then handle overflow with
    grid-auto-rows: 200px;
    or
    grid-auto-columns: 200px;

  • auto-fill and auto-fit

    • AUTO-FILL and AUTO-FIT are SUPER USEFUL!

    • Auto-placement keywords, like auto-fill + auto-fit, can be used to let the browser
      take care of the responsiveness of the grid and the wrapping of
      columns into new rows when needed.
    • auto-fill and auto-fit can be used as the first argument in the repeat() method
    • .container {
      display: grid;
      grid-template-columns: repeat(auro-fit, minmax(100px, 1fr));
      }
